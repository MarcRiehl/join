3. About the Application

"Join" is a web-based Kanban project management application designed to help users organize tasks and collaborate within teams.

The application provides features including:

  • Creating, editing, and deleting tasks
  • Organizing tasks by status
  • Assigning tasks to contacts
  • Managing contact information
  • Creating subtasks
  • Real-time synchronization between users

4. Personal Data We Process

Depending on how you use Join, the following personal data may be processed:

User account information

  • First name
  • Last name
  • Email address
  • Password information (processed securely through the authentication provider and not stored as plain text)

Contact information

  • First name
  • Last name
  • Email address
  • Phone number

Task-related information

  • Task titles
  • Task descriptions
  • Due dates
  • Priorities
  • Categories
  • Assigned contacts
  • Subtasks

Users may voluntarily enter additional information into text fields. Users should avoid entering unnecessary sensitive personal information.

5. Registration and Authentication

User registration and authentication are provided through Supabase Auth.

Users can access the application through:

  • Email and password registration
  • Guest / anonymous login

For guest access, no personal registration information is required.

Authentication data is processed to provide secure access to user accounts and application functionality.

6. Use of Supabase

Join uses Supabase as a Backend-as-a-Service provider.

Supabase provides:

  • Database hosting
  • Authentication services
  • Real-time data synchronization

Personal data processed through Supabase may include:

  • Account information
  • Contact information
  • Task data
  • Authentication data
  • Technical connection information such as IP address and timestamps

Data processing by Supabase is subject to Supabase's own privacy policies and security measures.

7. Database Storage

The application uses a PostgreSQL database provided by Supabase.

Stored information may include:

  • User profiles
  • Contacts
  • Tasks
  • Subtasks

This data is stored only for providing the application's functionality.

8. Real-Time Synchronization

Join uses Supabase Realtime services to synchronize changes to tasks and contacts between users.

Data updates are transmitted through secure connections to enable collaboration features.

9. Google Fonts and Material Icons

This application uses Google Fonts and Google Material Icons.

When these resources are loaded, a connection to Google's servers may be established.

During this process, technical information such as the user's IP address may be transmitted to Google.

For improved privacy compliance, self-hosting these resources is recommended.

10. Cookies and Local Storage

Join does not use cookies for advertising, marketing, or analytics purposes.

The application uses browser Local Storage through the Supabase client library to store authentication session information, such as access and refresh tokens.

This storage is required to keep users authenticated and provide application functionality.

11. Analytics and Tracking

Join does not use analytics or tracking services such as:

  • Google Analytics
  • Marketing pixels
  • Advertising trackers

12. Legal Basis for Processing

Personal data is processed based on the following legal grounds:

Article 6(1)(b) GDPR:
Processing necessary for providing the requested application services.

Article 6(1)(f) GDPR:
Processing necessary for maintaining security and technical functionality.

13. Data Security

The application uses HTTPS connections for communication between the client and Supabase services.

Authentication and password management are handled by Supabase Auth.

Security measures depend partly on the configuration of the Supabase project and database access rules.

14. Data Retention

Personal data is stored only for as long as necessary to provide the application's services.

Data will be deleted when it is no longer required, unless legal obligations require longer storage.
